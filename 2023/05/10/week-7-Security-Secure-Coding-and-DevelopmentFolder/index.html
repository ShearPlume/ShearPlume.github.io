JAVA标准加密库

实现我们自己的加密算法是一个坏主意。
➢ 建立一个安全的系统可能需要使用标准的加密算法。
➢ 我们通过使用所提供的加密算法库来做到这一点。
➢ Java 有自己的加密包：
▪ java.security.*
▪ javax.crypto..
▪ javax.crypto.spec..

密码库： 接口和实现

这些库是以标准方式实现的。
➢ 所有功能都在接口中提供。
➢ 实际的实现是由工厂方法提供的。
➢ 这使得切换实现很容易。
▪ 大部分代码使用接口，不会改变。只有工厂方法需要改变，以选择不同的实现。

文件加密应用程序： 实施

这个程序对一些小文件进行加密。
▪ 它们主要是文本文件。
➢ 用户提供加密密钥，并使用Java库来进行加密。
➢ 相关文件被分组在一个索引中，只产生一个加密文件。
➢ 该索引在加密前被压缩。
➢ 打开时产生明文文件。
➢ 明文文件在关闭或应用程序关闭时被擦除，然后被删除。

加密方案：128 x 128 AES。
➢ 密钥由用户提供，为一个字符串（<=16个字符）。
➢ 被转换为适合AES的128位密钥。
▪ 这是16个字节。
▪ 通过添加’z’字符将密钥填充为 16 个字符。
➢ Javax.crypto.spec有一个名为Key的抽象类（接口）。实现类SecretKeySpec用于创建一个实际的密钥对象。
▪ 第一个参数是一个 16 字节的数组，来自字符串。
▪ 第二个参数指定为 “AES”。
